Les options "drop X pour simplifier" sont presque toujours rejetées en faveur de "garder X mais le déplacer en accordéon / sélecteur / sous-section".

**Why:** Pattern confirmé sur 2 sessions consécutives :
- Session /portfolio (2026-05-14, premier choix narratif) : oscillation C → D → C par auto-correction sur outside voice. Pas de réduction réelle de scope.
- Session /historique refonte (2026-05-14, refonte) : D1 = "B+C" (dédoublonner + voix), D2 = "A+D" (narratif + grille), D3 = coche P2/P3/P4 fragiles donc REJET de "drop tableau Allocation" et "drop sélecteur mois" → garde tout en accordéon/déroulé, D4 = C ambitieux avec mini-embed.

Le signal : Boris fait confiance à l'ergonomie qu'il a construite. Il veut *mieux composer*, pas *moins afficher*. Le "boil the lake D" sur outside voice findings est cohérent (intégrer 10/10 trous, jamais skip).

**How to apply:** Quand on propose des alternatives en /office-hours sur DCATrack, biaiser les options vers "réorganiser/composer" et présenter "drop X" comme option C par défaut (pas A recommandée), sauf si X est clairement mort/inutile. Pour les findings outside voice, anticiper D = boil the lake (configurer la AUQ avec D recommended si pas sûr de la priorité).
